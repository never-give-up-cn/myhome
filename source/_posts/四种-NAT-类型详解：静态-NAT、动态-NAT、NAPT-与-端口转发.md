**NAT（Network Address Translation，网络地址转换）** 是解决 IPv4 地址短缺的核心技术之一，也是家庭和企业网络中几乎每天都在使用的功能。它允许内部网络使用私有 IP 地址（如 `192.168.x.x`、`10.x.x.x`），通过一个或少数几个公网 IP 访问互联网。

根据转换方式的不同，NAT 主要分为四种类型，下面逐一讲解。

<!-- more -->

---

## 一、为什么要用 NAT？

在 IPv4 协议设计之初，32 位地址空间提供了约 43 亿个地址，看似充裕，但随着互联网爆炸式增长，地址早已枯竭。NAT 的出现让大量设备可以共享少量公网 IP，极大延缓了 IPv4 耗尽的速度。

NAT 的核心作用：

- **地址复用**：整个局域网共用 1 个（或几个）公网 IP
- **安全隔离**：外部无法主动访问内网设备，天然形成一层防火墙
- **灵活组网**：内网 IP 可随意规划，不依赖公网分配

---

## 二、四种 NAT 类型

### 1. 静态 NAT（Static NAT）

**一对一映射**，一个内网 IP 固定对应一个公网 IP。

```
内部地址      ⇢     公网地址
192.168.1.10 ⇢    203.0.113.10
192.168.1.20 ⇢    203.0.113.11
```

**特点：**

- 内网 IP 与公网 IP 是**固定绑定**关系
- 一个公网 IP 只服务一个内网设备
- 外部可以通过公网 IP **主动访问**内网对应设备

**适用场景：**

- 内网服务器需要对外提供固定服务（Web、邮件、VPN）
- 企业对外暴露的关键业务系统
- 需要保证双向访问可达的场景

**优点：** 配置简单，双向通信无障碍  
**缺点：** 消耗公网 IP 地址，无法缓解地址短缺

---

### 2. 动态 NAT（Dynamic NAT / Pooled NAT）

**多对多映射**，内网 IP 从一个公网 IP 池中动态获取映射。

```
内网设备         ⇢    公网 IP 池
192.168.1.10    ⇢    203.0.113.10 （当前）
192.168.1.20    ⇢    203.0.113.11 （当前）
192.168.1.30    ⇢    等待空闲 IP
```

**工作流程：**

1. 内网设备发起对外访问时，路由器从公网 IP 池中选取一个空闲 IP
2. 会话结束后，公网 IP 释放回池中，供其他设备使用
3. 映射关系是临时的，不固定

**特点：**

- 公网 IP 数量决定**最大并发访问设备数**
- 当池中 IP 全被占用时，新设备无法上网（需等待释放）
- 外部无法主动访问内网（映射关系是临时建立的）

**适用场景：**

- 公网 IP 数量有限但多于 1 个的企业网络
- 员工上网、对外访问需求，不需要外部主动接入

**优点：** 一定程度上节省公网 IP  
**缺点：** 并发设备数受限于 IP 池大小；外部无法主动访问

---

### 3. NAPT（网络地址端口转换）

**多对一映射**，也是我们最常说的「端口映射」或 **PAT**（Port Address Translation）。成千上万个内网设备通过 **同一个公网 IP + 不同端口** 访问互联网。

```
内部地址                ⇢    公网地址（同一 IP）
192.168.1.10:12345     ⇢    203.0.113.1:30000
192.168.1.10:12346     ⇢    203.0.113.1:30001
192.168.1.20:54321     ⇢    203.0.113.1:30002
```

**核心原理：**

路由器维护一张 **NAT 转换表**，记录每个会话的：

| 内网 IP | 内网端口 | 公网 IP | 公网端口 | 目标 IP:端口 |
|---------|---------|---------|---------|-------------|
| 192.168.1.10 | 12345 | 203.0.113.1 | 30000 | 8.8.8.8:443 |
| 192.168.1.20 | 54321 | 203.0.113.1 | 30001 | 1.1.1.1:53 |

当外部返回数据包时，路由器根据 **目标端口** 查找转换表，还原成内网地址，完成转发。

**特点：**

- **一个公网 IP 可供整个局域网上网**
- 端口数量决定了最大并发会话数（理论上 65535 个端口，实践中约几万个）
- 家庭宽带几乎都使用 NAPT（光猫/路由器默认模式）

**适用场景：**

- 家庭宽带上网（99% 的家庭路由器运行 NAPT）
- 公司内网通过单一公网出口访问互联网
- 学校、酒店等大规模局域网共享出口

**优点：** 最大程度节省公网 IP，每个家庭只需 1 个 IP  
**缺点：** 外部无法主动访问内网；大量 P2P 应用（游戏、BT 下载）可能遇到兼容性问题

---

### 4. 端口转发（Port Forwarding）

严格来说端口转发不是 NAT 的一种独立类型，而是 **NAPT 的补充机制**。它手动在路由器上创建一条静态规则，告诉路由器：「某个公网端口的流量，请转发给内网某台设备的某个端口」。

```
规则：公网 203.0.113.1:3389 → 内网 192.168.1.100:3389
规则：公网 203.0.113.1:8080 → 内网 192.168.1.200:80
```

**为什么需要端口转发？**

NAPT 默认只允许内网→外网的主动连接。如果想让外网访问内网的服务器（NAS、Web、远程桌面），就必须手动配置端口转发。

**配置示例（路由器后台典型界面）：**

| 服务名称 | 外部端口 | 内网 IP | 内部端口 | 协议 |
|---------|---------|---------|---------|------|
| Web 服务器 | 80 | 192.168.1.100 | 80 | TCP |
| RDP 远程桌面 | 3389 | 192.168.1.100 | 3389 | TCP |
| Minecraft 游戏 | 25565 | 192.168.1.200 | 25565 | TCP |

**特点：**

- 需要**手动配置**，每条规则对应一个服务
- 可以配合 DDNS（动态域名解析）使用，解决公网 IP 变动问题
- 安全性需注意：暴露端口等于开放了攻击面

**适用场景：**

- 外网访问家里的 NAS（群晖、威联通）
- 搭建个人网站、游戏服务器
- 远程桌面连接办公电脑

**优点：** 灵活控制外部访问，按需开放端口  
**缺点：** 配置繁琐；暴露端口有安全风险；家庭宽带可能被运营商封禁 80/443 等端口

## 三、四种 NAT 类型对比一览

| 特性 | 静态 NAT | 动态 NAT | NAPT (PAT) | 端口转发 |
|------|---------|---------|-----------|---------|
| 映射方式 | 1:1 固定 | N:N 动态 | N:1 端口 | N:1 固定规则 |
| 公网 IP 消耗 | 每个设备 1 个 | 池化分配 | 1 个够全家 | 复用出口 IP |
| 外部主动访问 | ✅ 支持 | ❌ 不支持 | ❌ 不支持 | ✅ 按规则支持 |
| 配置复杂度 | 低 | 中 | 低（默认） | 中 |
| 适用规模 | 企业服务器 | 中型企业 | 家庭/小型办公 | 个人/家庭服务 |
| IPv4 地址节约 | ❌ 不节约 | ⚠️ 有限节约 | ✅ 最大节约 | ✅ 最大节约 |

## 四、实际生活中的 NAT

以典型家庭网络为例：

```
光猫（NAPT）
  │
  └── 路由器（NAPT + 端口转发）
        ├── 手机/平板（通过 NAPT 上网）
        ├── 电脑（通过 NAPT 上网）
        ├── NAS（通过端口转发 5000/5001 外网访问）
        └── 游戏机（通过 NAPT + UPnP 自动开端口）
```

- **手机、平板、电脑**：通过 NAPT 正常上网，无感
- **NAS 远程访问**：配置端口转发 + DDNS
- **Switch/PS5 联机游戏**：依赖 UPnP 自动配置 NAT 规则，或手动开端口
- **BT 下载**：需要端口转发以获得 High ID，提升下载速度

---

## 五、NAT 的局限与未来

尽管 NAT 功不可没，但它并非完美：

- **破坏端到端原则**：互联网设计的初衷是任意两点可直接通信，NAT 打破了这一点
- **P2P 通信困难**：内网设备之间无法直接建立连接，需要 STUN/TURN/ICE 等穿透技术
- **协议兼容性问题**：某些应用层协议（如 FTP、IPsec）在 NAT 环境下需要特殊处理（ALG）
- **增加延迟**：每个数据包都需经过 NAT 转换，增加处理开销

**IPv6 的到来**将从根本上解决这些问题——IPv6 拥有足够多的地址，每个设备都可以获得全球唯一的公网 IP，不再需要 NAT。然而，由于 IPv4 的广泛部署，NAT 在可预见的未来仍将长期存在。

---

## 六、小结

- **静态 NAT**：一对一固定映射，适合对外提供服务的服务器
- **动态 NAT**：从 IP 池中动态分配，适合公网 IP 较多的企业
- **NAPT（PAT）**：最常见的家庭上网方式，全家人共享一个公网 IP
- **端口转发**：NAPT 的补充，让外网可以访问内网服务

理解这四种 NAT 类型，能帮你更好地排查网络问题、配置家庭服务器，以及理解互联网的工作原理。
